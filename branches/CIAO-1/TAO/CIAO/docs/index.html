CIAO - Component-Integrated ACE ORB

Introduction

CIAO is TAO's implementation of CORBA Component Model (CCM) specially optimized for distributed real-time embedded (DRE) systems. This page contains links to other documents and resources for using CIAO.

Documents

Project Status

Modules Status
TAO_IDL TAO_IDL front-end has been extended to handle the keyword extension added by compnent model specification. (It is still in Jeff's branch, however.) The next stage will be to start implementing the code for backend code generation.
CIAO_CIDL CIAO_CIDL is still an unknown at this point. We do know that we have no plan to support persistentstate service in the near future. For starter, we will probably only support a simple construct like the followingsee : 
        import <some_component.idl>;

        module component_composit_nasee me {
          composition session component_home_executor {
            implements component_home;
            manages component_executor;
          };
        };
We will need to generate the implementation for the following interfaces/operations (also see this example ):
  • Navigation
  • Receptacle
  • Event
  • Internal/callback interface?
  • Executor stuff?
CCM Core We are currently working on the prototype that can install a component remotely using the CCM ComponentInstallation interface. Since TAO_IDL still doesn't generate component specific code yet, and we still don't have a CIDL implementation, we will also write up an example to show how the code should be generated in both TAO_IDL and CIAO_CIDL compiler.
