Current Status

Modules Status
CCM Core We are currently working on a prototype that enables remote component installation using the CCM ComponentInstallation interface. Since we still don't have a CIDL implementation, we will also write up an example to show how the code should be generated in both CIAO_CIDL compiler and provide example component implementations.
TAO_IDL TAO_IDL has been extended to handle and generate all the component related keyword but import. Although the generated code has not been tested extensively. TAO_IDL Check out TAO IDL compiler for more details.
CIAO_CIDL We still have no solid schedule for implementing CIAO_CIDL at this point and we do not have immediate plan to integrate Persistence State Service with CIDL. For starter, we will probably only support a simple construct like the following: 
        import ::some_idl_module;

        module component_composite_name {
          composition session component_home_executor {
            implements component_home;
            manages component_executor;
          };
        };
We will need to generate the implementation for the following interfaces/operations (also see this example ):
  • Navigation
  • Receptacle
  • Event
  • Internal/callback interface?
  • Executor stuff?

TO-DOs

 This to-do list is currently arranged in chronological order, i.e., we group features that are expected to be available in certain release together. All interfaces and types are under Components module namespace if not specified explicitly.
  • CIAO 0.1 alpha release - (TAO 1.3, Oct. 2002)
    • Default implementation of
      • Navigations
      • Receptacles
      • Events
      interfaces and their supporting data types that CCMObject interface requires. These interfaces are defined in       CIAO_Base.pidl and CIAO_Event.pidl.
    • Default implementation of
      • CCMObject
      • CCMHome
      • KeylessCCMHome
      and their supporting data types as defined in part of the CIAO_Component.pidl .
    • Partial support for basic container/callback interfaces:
      • CCMContext, SessionContext
      • EnterpriseComponent, SessionComponent
    • Extension to CORBA Core
      • CORBA::LocalObject interface has been added a long time ago.
      • get_component() method in CORBA::{Object|LocalObject} interfaces.
    • HomeExecutorBase interface
    • Support (hand crafted) monolithic executor implementation
    • Extension to IDL compiler and Typecode factory. IDL compiler shall recognize all CCM keyword extensions but not necessarily generate code for these added keywords.
    • Adaptive configuration of Notification Service/Event Channels
    • Partial implementation of Deployment module.
    AT this stage, we only implement "enough" CCM defined interface capability to allow manual implementation of components and a simple framework that offer some remote installation/composition capability. This will help design how CIDL generated code should look like.
