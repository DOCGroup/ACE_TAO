Database Support for CUTS

System Requirements

The following are the miminal system requirements for installing database support for CUTS.

  • MySQL 5.0
Windows Environment(s)
  • MySQL Connector/ODBC 3.51
Linux/Unix Environment(s)
  • UnixODBC

Installing Database and Drivers

Installing MySQL 5.0

There are no special requirements for installing MySQL 5.0. However, we do require you to remember the username and password for root access. If you do not remember these, then you will not be able to install the CUTS database.

Installing MySQL Connector/ODBC 3.51

There are no special requirements for installing MySQL Connector/ODBC 3.51 Downloads. Just follow the instructions that come with the driver.

Installing the CUTS Database

The following steps are necessary for installing the CUTS database. We are assuming you have successfully installed MySQL 5.0. The installation scripts are located in $CUTS_ROOT/SQL.

  1. %> mysql --user=root [--password=<root-password>] < CUTS-create.sql
  2. %> mysql --user=root [--password=<root-password>] < CUTS-procs.sql

Configuring CUTS Data Source Name (DSN)

CUTS uses Data Source Names (DSNs) to communicate with its database. The following are steps for configuring the DSN for your platform.

Windows Environment(s)

  1. Open the Data Sources (ODBC) tool. On Windows 2000 and above, it's located in: Control Panel | Adminstrative Tools | Data Sources (ODBC)
  2. In the User DSN tab, select Add... If you want to make the DSN system-wide, do this under the System DSN tab.
  3. Scroll down the listing, select MySQL ODBC 3.51 Driver, and click the Finish button.
  4. In the Login tab fill in the following table:
    Data Source Name: CUTS
    Description: CUTS Performance Metrics Database
    User: cuts
    Password: cuts
    Database: cuts
  5. Test the connection by clicking the Test button.
  6. Click the OK button to finish and exit.
