Supported Platforms for CUTS

CUTS should be usable on all platforms that support CIAO. Please see CIAO's installation page for details.

The modeling aspect of CUTS is supported on all platforms that support CoSMIC. Please see CoSMIC's installation page for details.

Building and Installing CUTS

The first thing you should do to build & install CUTS is to set the environment variable CUTS_ROOT=$COSMIC_ROOT/CUTS. Please refer to the CIAO installation notes and CoSMIC installation notes for instructions on setting $COSMIC_ROOT and $CIAO_ROOT, respectively. They are needed since CUTS sits on top of ACE+TAO+CIAO and uses CoSMIC. Compiling ACE+TAO+CIAO and CoSMIC successfully is the first requirement for compiling CIAO.

CUTS is divided into several parts, so different installation/building rules apply to different parts of CUTS. Each of these parts is built and installed by enabling features at compile time.

To determine which features are needed for your build of CUTS, please refer to $CUTS_ROOT/default.features. Each feature is a boolean flags that takes the value 0 or 1. To enable a feature, set the value equal to 1. For example, if you wanted to build the modeling features of CUTS, you would do the following in default.features.

mfc = 1
udm = 1
boost = 1

To build CUTS, we use the Makefile, Project, Workspace Creator (MPC) utility. Please see the MPC page for more details. Run MPC on the $CUTS_ROOT/CUTS.mwc using the following command:

$ACE_ROOT/bin/mwc.pl [command-line args] CUTS.mwc
Notice: Type $ACE_ROOT/bin/mwc.pl --help for help with MPC.

After MPC is complete, you can build the CUTS framework and any enabled features. For example, if you are using gcc, you would use the following command:

gmake

Whereas Visual Studio compilers will open the respective solution file and click the build command.

Note for Non-Windows Users

If you have enabled any of the features in default.features, you must enable the features as a macro on the command-line when building CUTS. For example, if you enabled unixodbc = 1, your build command should be as follows:

gmake unixodbc=1

Failure to enable the feature at the command-line will result in any projects that require that feature to not build.
