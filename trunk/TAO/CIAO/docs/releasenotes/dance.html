Deployment And Configuration Engine (DAnCE)
Release Notes

Deployment And Configuration Engine (DAnCE) is an implementation of the run-time infrastructure as mandated by the recently adopted OMG Deployment & Configuration (D&C) specification, which deals with the configuration and deployment of component-based applications. Currently DAnCE only supports the deployment & configuration of CORBA Component Model (CCM) based applications. DAnCE needs XML descriptors which drive the deployment to be provided to it, to actually perform the deployment. One way to generate these descriptors is using the Component Synthesis using Model-Integrated Computing (CoSMIC) tool-suite. DAnCE uses Component Integrated ACE ORB (CIAO) is used as the underlying CCM implementation in the current release.

D&C XML Schema Handling

The XML handlers shipped with DAnCE are capable of handling almost all elements that are defined in the schema. Handling here refers to reading the XML tags and converting them to the equivalent in-memory IDL based data structures defined by the specifications. Some of the elements are however not handled. The list of XML elements that are not supported at this time are listed below:

  • Deployment:Any — This is a complex element which is composed of many sub-elements. Currently, DAnCE only parses a subset of the sub-elements of Deployment:Any. Specifically, DAnCE supports all the basic data types defined as part of Deployment:Any, and doesn't understand any of the complex types.
  • xmi:Extension — This element is designed to allow augmenting the standard D&C schema with user-defined elements. Currently DAnCE doesn't support user-defined extensions to the schema.
  • Deployment:Requirement — This element is designed to allow description of requirements of component-based applications. Since handling this requires interaction with the TargetManager interface(which is unsupported at this time), support for handling it is not present.
  • Deployment:ComponentPropertyDescription — This element is used to specify default values for attributes of component. DAnCE doesn't support setting the values for attributes of components using this tag.
  • Deployment:ComponentExternalPortEndpoint, Deployment:ExternalReferenceEndpoint — These elements are used specify connections between ports of components in one assembly with ports of components in a different assembly, or connections between component ports and external connection endpoints. These features are needed for supporting assemblies of components. DAnCE doesn't support hierarchical assemblies i.e., creation of assemblies which can themselves contain sub-assemblies.
  • Deployment:PlanPropertyMapping — This element is used to delegate property mapping between a component's external properties to properties of subcomponents that actually implement the behavior of the larger component. Since DAnCE doesn't support hierarchical assemblies, this tag is not supported currently.

D&C Management Interfaces

D&C specification defines a set of standard CORBA interfaces which are used to carry out the deployment of component-based applications. These interfaces depend on the in-memory IDL data structures that are built by the XML handlers. The following lists the interfaces defined by the specification and the level of support for each interface in DAnCE:

  • RepositoryManager — Only the installPackage() and findPackageByName() operations of this interface are implemented.
  • TargetManager — This interface has not been implemented at all. Implementation of this interface is a planned activity for the future.
  • ExecutionManager — All operations defined in this interface are implemented.
  • DomainApplicationManager, DomainApplication — All operations except getApplications() operation of DomainApplicationManager are implemented.
  • NodeManager, NodeApplicationManager, NodeApplication — All operations defined in these interfaces are implemented.
