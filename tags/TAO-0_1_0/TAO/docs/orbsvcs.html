TAO Services Directory Hierarchy

The file and directory organization for TAO services can be confusing at first glance (and even on subsequent ones to be honest), so we felt like some rationale and explanation of the directory hierarchy was needed.

For general sanity all TAO services files are located under $TAO_ROOT/orbsvcs. The current structure for this directory emphasizes the decoupling from the interfaces (stubs) and their implementation; for this reason the services implementations are each on a separate directory, for instance Naming Service is in $TAO_ROOT/orbsvcs/Naming_Service.

It is expected that clients use more than one service at the same time (in fact some of the services already do, for instance the Event Service uses the Naming Service and the Scheduling Service); for this reason all the services stubs are grouped in one library; this library is located in $TAO_ROOT/orbsvcs/orbsvcs. Usually the include path is only $TAO_ROOT/orbsvcs, so files are included like in:

#include "orbsvcs/CosNamingC.h"

To simplify the IDL generation the skeletons are also on the library, this is not a problem for client programs and most services need to link the library anyway (since they use other services.)

Finally some test are located in $TAO_ROOT/orbsvcs/tests, since most tests require several files each one has its own directory.
